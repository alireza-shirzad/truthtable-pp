\section{PIOPs for Strings}


\subsubsection{Activator Consistency Check}
\label{sec:act-consistency}
The \emph{Activator Consistency Check} takes the source column $\mathsf{src}$, the data activator $\mathsf{a_d}$, the length column $\mathsf{l}$, and the string activator $\mathsf{a_h}$, and proves that $\mathsf{a_d}$ and $\mathsf{a_h}$ are consistent: for every string, the number of its active characters equals its length when the string is active, and is $0$ otherwise. Equivalently, for every string index $i$,
\begin{displaymath}
  \mathsf{a_h}[i]\cdot\mathsf{l}[i] \;=\; \sum_{j:\,\mathsf{src}[j]=i} \mathsf{a_d}[j].
\end{displaymath}
We package this as the relation
\begin{displaymath}
\NPRelation_{\mathsf{ActCons}} = \left\{\,
  \NPInstance = \bot,\;
  \NPInstancey = (\Oracle{\MLE{\mathsf{src}}}, \Oracle{\MLE{\mathsf{a_d}}}, \Oracle{\MLE{\mathsf{l}}}, \Oracle{\MLE{\mathsf{a_h}}}),\;
  \NPWitness = (\mathsf{src}, \mathsf{a_d}, \mathsf{l}, \mathsf{a_h})
  \;\middle|\;
  \forall i,\; \mathsf{a_h}[i]\cdot\mathsf{l}[i] = \!\!\!\sum_{j:\,\mathsf{src}[j]=i}\!\!\! \mathsf{a_d}[j]
\,\right\}.
\end{displaymath}
The check is exactly a group-by-sum: grouping the rows $(\mathsf{src}, \mathsf{a_d})$ by $\mathsf{src}$ and summing $\mathsf{a_d}$ within each group must reproduce $\mathsf{l}$ on the active strings (those marked by $\mathsf{a_h}$). Treating $(\mathsf{src}, \mathsf{a_d})$ as an input table grouped by its key and $(\mathsf{ind}, \mathsf{l})$ with activator $\mathsf{a_h}$ as the grouped output, this is discharged by a single \cref{piop:keyedsumcheck}.
\begin{DefinePIOP}{Activator Consistency Check}{act-cons}
  \emph{Inputs:} $\Oracle{\MLE{\mathsf{src}}}, \Oracle{\MLE{\mathsf{a_d}}}, \Oracle{\MLE{\mathsf{l}}}, \Oracle{\MLE{\mathsf{a_h}}}$.
  \begin{enumerate}[nosep]
    \item $\PIOPProver$ and $\PIOPVerifier$ locally form the input table $\Left$ with key column $\Left.\key = \mathsf{src}$, value column $\Left.\val = \mathbf{1}$, and activator $\Left.\act = \mathsf{a_d}$ (so the keyed sum counts the active characters of each string).
    \item $\PIOPProver$ and $\PIOPVerifier$ locally form the output table $\Right$ with key column $\Right.\key = \mathsf{ind}$ (the index column, $\mathsf{ind}[i] = i$), value column $\Right.\val = \mathsf{l}$, and activator $\Right.\act = \mathsf{a_h}$.
    \item $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:keyedsumcheck} on $\Left$ and $\Right$, which forces $\sum_{j:\,\mathsf{src}[j]=i}\mathsf{a_d}[j] = \mathsf{a_h}[i]\cdot\mathsf{l}[i]$ for every key $i$.
  \end{enumerate}
\end{DefinePIOP}

    \subsubsection{Prefix and Suffix Check} A prefix/suffix check keeps the strings in a column that begin or end with a fixed pattern $\mathsf{str}$ of length $k$: a \emph{prefix} filter $\mathsf{str}\%$ keeps strings that start with $\mathsf{str}$ (as in \texttt{LIKE 'str\%'}), and a \emph{suffix} filter $\%\mathsf{str}$ keeps strings that end with it (as in \texttt{LIKE '\%str'}).

    \parhead{What the protocol proves}The verifier is given a tuple $( \mathsf{h},\mathsf{b},\mathsf{l},\mathsf{a_h^{old}}, \mathsf{d}, \mathsf{src}, \mathsf{a_d^{old}}, \mathsf{s_b})$ and new activators $\mathsf{a_h^{new}}$ and $\mathsf{a_d^{new}}$. The prover convinces the verifier that the tuple $( \mathsf{h},\mathsf{b},\mathsf{l},\mathsf{a_h^{new}}, \mathsf{d}, \mathsf{src}, \mathsf{a_d^{new}}, \mathsf{s_b})$ is the result of applying the filter to the original tuple.


At a high level the check has five parts, which we explain in turn before presenting the protocol. For concreteness we describe the prefix scenario throughout; afterwards we show how the prefix and suffix protocols can be unified into a single check.


\parhead{Activator consistency} The prover claims that $\mathsf{a_h^{new}}$ and $\mathsf{a_d^{new}}$ are the activators of the filtered column. The verifier checks that they are internally consistent with one another and with the lengths: a string is marked active by $\mathsf{a_h^{new}}$ exactly when its characters are marked active by $\mathsf{a_d^{new}}$, and the active-character count of each string equals its length.

\parhead{Filtering by length} A string shorter than $k$ can never match a length-$k$ pattern, and leaving it in place is actively harmful: once the anchored selectors are rotated inward, a short string's selectors run past its own end and spill into the character slots of its neighbors, corrupting the comparison. The first step therefore removes every string of length less than $k$. The prover flags the short strings with two masks: $\mathsf{m_h}$, set to $1$ on each string of length less than $k$, and $\mathsf{m_d}$, set to $1$ on every character of such a string. Afterwards, an activator consistency check (\cref{sec:act-consistency}) ties $\mathsf{m_h}$ and $\mathsf{m_d}$ together, forcing a string to be flagged by $\mathsf{m_h}$ exactly when all of its characters are flagged by $\mathsf{m_d}$. Two sign checks then certify that $\mathsf{m_h}$ flags \emph{exactly} the short strings: a \emph{Negative} sign check over the flagged rows forces every flagged string to satisfy $\mathsf{l}-k<0$, i.e.\ length below $k$, while a \emph{Non-Negative} sign check over the unflagged rows forces every unflagged string to satisfy $\mathsf{l}-k\geq0$, i.e.\ length at least $k$. Together these leave the prover no room to misflag a string in either direction --- it can neither smuggle a long string out of the filter nor hide a short one inside it. With the masks verified, the parties switch the short strings off by defining the masked activators
\begin{displaymath}
  \mathsf{a_d^{msk}} := \mathsf{a_d^{old}}\cdot(1-\mathsf{m_d}), \qquad
  \mathsf{a_h^{msk}} := \mathsf{a_h^{old}}\cdot(1-\mathsf{m_h}),
\end{displaymath}
and use these in place of the originals throughout, so that every later step ranges only over strings long enough to be candidate matches.



\parhead{False positives Check} A string that \emph{survives} the filter must genuinely match the pattern. To enforce this, the prover and verifier walk through the $k$ pattern positions and, at each one, prove that every surviving string holds the required pattern character there.

\parhead{False negatives Check} Conversely, a string that the prover \emph{drops} must truly fail the pattern; otherwise a legitimate match would be silently discarded. The prover first commits, for each position $i$, an equality predicate $\mathsf{p}_i$ that marks the character slots where the data matches $\mathsf{str}[\pi(i)]$ at an active, anchored position. Two checks pin $\mathsf{p}_i$ to the truth: a zero-check forces $\mathsf{p}_i$ to vanish wherever the characters disagree, and a no-zero check forces it to be nonzero wherever an active anchored slot does agree, so $\mathsf{p}_i$ marks precisely the positional matches. To test whether a string matches at \emph{every} one of the $k$ positions, these predicates---which live at $k$ different anchored offsets---are rotated back onto the common anchor by $\mathsf{p}_i' := \rho_{-\epsilon i}(\mathsf{p}_i)$, with each rotation certified by a rotation check, so that their product $\prod_i \mathsf{p}_i'$ is $1$ exactly at the anchor of a string matching the entire pattern.

\parhead{Dropped strings really fail} Finally, the dropped-anchor selector $\mathsf{s_a'} := (1-\mathsf{a_d^{new}})\cdot\mathsf{a_d^{msk}}\cdot\mathsf{s_a}$ marks the anchor of every string that was active before the filter but dropped after it. A single zero-check on $(\prod_i \mathsf{p}_i',\, \mathsf{s_a'})$ forces $\prod_i\mathsf{p}_i'$ to vanish at each dropped anchor, certifying that no dropped string matched the full pattern. Together with the false-positive check, this pins the new activators to keep exactly the strings that match the filter.

\parhead{Unifying prefix and suffix} The prefix and suffix cases differ only in which end of each string the comparison is anchored to. Let $\rho_t(\cdot)$ denote rotation of a column by $t$ positions (a $1$ at index $p$ moves to index $p+t$), and let $\epsilon := +1$ for a prefix filter and $\epsilon := -1$ for a suffix filter. Define the \emph{anchor selector}
\begin{displaymath}
  \mathsf{s_a} := \begin{cases} \mathsf{s_b} & \text{prefix},\\[2pt] \rho_{-1}(\mathsf{s_b}) & \text{suffix,}\end{cases}
\end{displaymath}
which marks the first (resp.\ last) character of every string; the suffix anchor is $\mathsf{s_b}$ rotated one step backwards, landing on the character just before each boundary. For $i=0,\ldots,k-1$, the $i$-th anchored selector and the pattern character the filter requires there are
\begin{displaymath}
  \mathsf{s_a^i} := \rho_{\epsilon i}(\mathsf{s_a}), \qquad \mathsf{str}[\pi(i)], \quad \pi(i) := \tfrac{(k-1)(1-\epsilon)}{2} + \epsilon\, i,
\end{displaymath}
so $\pi(i)=i$ for a prefix and $\pi(i)=k-1-i$ for a suffix. The prefix case recovers $\mathsf{s_a}=\mathsf{s_b}$ and $\mathsf{s_a^i}=\mathsf{s_b^i}$.


   \begin{DefinePIOP}{Prefix/Suffix Check}{prefix-check}
    \begin{enumerate}[nosep]
        \item (\textit{Activator Consistency}) $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:act-cons} on $(\mathsf{src}, \mathsf{a_d^{new}}, \mathsf{l}, \mathsf{a_h^{new}})$.
        \item  (\textit{Filtering by length}) $\PIOPProver$ sends $\mathsf{m_h}$ and $\mathsf{m_d}$
        \begin{enumerate}[nosep]
          \item $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:act-cons} on $(\mathsf{src}, \mathsf{m_d}, \mathsf{l}, \mathsf{m_h})$.
        \item $\PIOPProver$ and $\PIOPVerifier$ invoke the Negative \cref{piop:sign-check} on the column $(\mathsf{m_h}, \mathsf{l}-k)$.
        \item $\PIOPProver$ and $\PIOPVerifier$ invoke the Non-Negative \cref{piop:sign-check} on the column $(1-\mathsf{m_h}, \mathsf{l}-k)$.
                \item $\PIOPProver$ and $\PIOPVerifier$ define the masked old activators
        \begin{align*}
          \mathsf{a_d^{msk}} := \mathsf{a_d^{old}}\cdot(1-\mathsf{m_d}), \qquad
          \mathsf{a_h^{msk}} := \mathsf{a_h^{old}}\cdot(1-\mathsf{m_h}),
        \end{align*}
        \end{enumerate}
        \item (\textit{False Positive Check}) For $i=0,\ldots,k-1$, $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:zerocheck} on the column $(\mathsf{d}-\mathsf{str}[\pi(i)],\;\mathsf{s_a^i}\cdot \mathsf{a_d^{new}}\mathsf{a_d^{msk}})$.
        \item (\textit{False Negative Check}) $\PIOPProver$ and $\PIOPVerifier$ proceed as follows:
        \begin{enumerate}
          \item (\textit{k independent equality expressions}) For $i=0,\ldots,k-1$, $\PIOPProver$ computes the equality predicate vector $\mathsf{p}_i$ such that
       \begin{align*}
        \mathsf{p}_i[j]= \begin{cases}
        1 & \text{if } \mathsf{d}[j]=\mathsf{str}[\pi(i)] \text{ and } \mathsf{s_a^i}[j]=1 \text{ and } \mathsf{a_d^{new}}[j]=1 \text{ and } \mathsf{a_d^{msk}}[j]=1\\
        0 & \text{otherwise}
        \end{cases}
       \end{align*}
       \begin{enumerate}[nosep]
       \item For $i=0,\ldots,k-1$, $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:zerocheck} on the column $(\mathsf{d}-\mathsf{str}[\pi(i)],\;\mathsf{p}_i)$.
        \item For $i=0,\ldots,k-1$, $\PIOPProver$ and $\PIOPVerifier$ invoke \cref{piop:nozero-check} on the column $(\mathsf{d}-\mathsf{str}[\pi(i)],\;(1-\mathsf{p}_i)\cdot \mathsf{s_a^i} \cdot \mathsf{a_d^{new}}\mathsf{a_d^{msk}})$.
        \item For $i=1,\ldots,k-1$, $\PIOPProver$ aligns each predicate back onto the anchor by setting $\mathsf{p}_i' := \rho_{-\epsilon i}(\mathsf{p}_i)$ and sends it to $\PIOPVerifier$; the parties then invoke \cref{piop:rotcheck} on $(\mathsf{p}_i, \mathsf{p}_i', -\epsilon i)$ to certify the rotation was performed correctly. Index $0$ needs no rotation, so $\mathsf{p}_0'=\mathsf{p}_0$.
        \end{enumerate}
        \item $\PIOPProver$ and $\PIOPVerifier$ define the dropped-anchor selector
          $\mathsf{s_a'} \;:=\; (1-\mathsf{a_d^{new}})\cdot \mathsf{a_d^{msk}}\cdot \mathsf{s_a}$
        and perform a \cref{piop:zerocheck} on the following column:
          $\prod_{i=0}^{k-1}\mathsf{p}_i',\;\mathsf{s_a'}$
        \end{enumerate}
    \end{enumerate}
\end{DefinePIOP}
